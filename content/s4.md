## Pourquoi cette session existe

> **Hook** : Tu as conçu un questionnaire sur papier. Maintenant il faut le faire vivre dans le vrai monde : mettre les 4 conditions dans Qualtrics, programmer la logique d'affichage, randomiser les stimuli, et exporter les données proprement pour SPSS. C'est la session "passage à l'acte".

**Idée centrale** : Qualtrics est l'outil qui transforme ton design expérimental (between-subjects, 4 conditions) en un vrai questionnaire en ligne. Chaque réglage a une raison méthodologique.

## 1. Rappel du design WWF [slides 1-3]

Le projet Matrix utilise un **design between-subjects** : chaque participant voit **une seule** des 4 campagnes WWF (Panda / Deer / Rain Forest / Rhino). L'objectif est de mesurer l'effet du visuel sur culpabilité, responsabilité et générosité.

| Variable | Rôle | Type |
|---|---|---|
| Condition (campagne) | IV | Nominale (4 groupes) |
| Genre | IV | Nominale |
| Âge | IV | Métrique |
| Culpabilité | DV | Métrique |
| Responsabilité | DV | Métrique |
| Don | DV | Métrique |

> **💡 Intuition** : Between-subjects = chaque participant est dans **un seul** groupe. Pas de contamination entre conditions. Mais il faut plus de participants (4 × 30 = 120 minimum).

## 3. Types de questions Qualtrics [slides 10-20]

### Multiple Choice
- Question fermée avec plusieurs options
- Idéal pour : genre, conditions, questions dichotomiques (keep/donate)
- Options : Single answer ou Multiple answers

### Matrix Table (tableau Likert)
- Plusieurs items sur une même échelle (ex. culpabilité : 5 items sur Strongly disagree → Strongly agree)
- Avantage : améliore le flux du questionnaire
- ⚠️ **Piège overuse** : trop de matrices décourageant les participants, surtout sur smartphone
- Utiliser quand les items sont formulés de façon similaire

### Slider
- Réponse numérique sur une plage (ex. 0–10 euros)
- Avantage vs text entry : donne un **ordre de grandeur** au participant
- Mobile-friendly si activé
- Configuration : minimum, maximum, nombre de décimales, grid lines

### Text Entry (Numeric Entry)
- Question type = Text entry → Text type = Single line
- Pour saisie numérique : activer **Add validation** → Content type = **Number** (avec min/max)
- Exemples : âge, montant précis

> **💡 Intuition** : Slider = "quelque part entre 0 et 10" (ancre le participant). Text entry number = le participant sait exactement combien il donne.

## 4. Structure complète du questionnaire WWF [slides 21-40]

```
Block 1: Informed Consent
├── Texte de consentement (anonymat, durée ~10 min)
├── "Yes I agree. Let's continue!" → continue
└── "No I do not agree" → Skip logic → End of Survey

Block 2: Between-picture (randomisation)
├── 4 pages (une par condition), chacune avec son image WWF
└── Question randomization → Present only 1 of total questions

Block 3: Guilt & Responsibility (Matrix Table)
├── 4 items culpabilité (I feel guilty about the situation...)
└── 4 items responsabilité (I feel responsible for...)
   Scale: Strongly disagree / Somewhat disagree / Neither / Somewhat agree / Strongly agree

Block 4: Behavioral DV – Donation
├── Q10 (MCQ) : "I decide to keep the amount" / "I decide to donate a portion of the amount to WWF"
└── Q11 (Slider 0-10€) : "How much of the amount do you want to donate?"
   → Display logic : afficher Q11 SEULEMENT si Q10 = "I decide to donate" Is Selected

Block 5: Demographics
├── Age (Text entry, numeric, required)
├── Gender (MCQ : Male / Female / Non-binary / Prefer not to say)
└── Donation covariate : "Are you currently a donor?" (3 options)

Block 6: Debriefing & End
└── "This questionnaire is part of a larger survey on visual campaigns. You took part in an experimentation."
```

> **💡 Intuition** : La logique d'affichage (Display Logic) est une condition `if/then` : si Q10 = "donate" → montrer Q11. Sans ça, tous les participants verraient le slider même s'ils ont refusé de donner.

## 5. Fonctionnalités Qualtrics avancées [slides 22-33]

### Question Randomization
- **Où** : cliquer sur le **block** (pas la question) → Block behavior → Question randomization
- **Réglage** : "Present only **1** of total questions" → Qualtrics choisit aléatoirement 1 condition parmi 4
- Résultat : chaque participant voit une seule image WWF

> **⚠️ Attention** : Activer la randomisation **désactive** le page break, skip logic et l'ordre des questions à l'intérieur du block.

### Display Logic
- **Où** : sur la question à conditionner → Question behavior → Display logic
- **Réglage** : "Display this question only if [Question précédente] [réponse] Is Selected"
- Exemple WWF : Q11 (slider) → Display if Q10 → "I decide to donate a portion" → Is Selected

### Skip Logic
- Sur la question de consentement : Skip to "End of Survey" if "No I do not agree" Is Selected
- Implémenté sur le block Informed Consent

### Collaborate
- **Où** : Tools → Collaborate
- **Obligatoire** pour le projet de groupe : permet aux coéquipiers d'accéder et modifier le survey

### Publication
1. Preview (toujours tester avant de publier !)
2. Publish → Expert Review (check qualité automatique) → Publish
3. Survey activated → **Copy link** → partager aux participants

## 6. Export des données [slides 31-33]

**Chemin** : Data & Analysis → Export & Import → Export Data

| Option | Réglage recommandé |
|---|---|
| Format | **CSV** ou **SPSS** direct |
| Numeric responses | **Use numeric values** (pas "choice text") |
| Compress as .zip | ✅ |
| Recode unanswered as | **-99** (seen but unanswered) |
| Multi-value fields | **0** |
| Export viewing order | ✅ (pour surveys randomisés) |

> **⚠️ Piège** : Exporter en "choice text" (ex. "Male") complique l'analyse SPSS. Toujours **numeric values** pour que SPSS ait des chiffres.

> **💡 Intuition** : Le code -99 pour les questions vues mais non répondues te permet de les distinguer des questions jamais montrées (missing data) dans SPSS.

## ✅ Test toi sur S4

**Q1** — Dans Qualtrics, comment s'assure-t-on que chaque participant voit UNE SEULE condition sur 4 ?

> **Réponse** : On met les 4 images dans le **même block**, puis on active **Question Randomization → Present only 1 of total questions**.

**Q2** — Comment faire apparaître le slider de donation (Q11) seulement si le participant choisit de donner (Q10) ?

> **Réponse** : **Display Logic** sur Q11 : "Display this question only if Q10 → 'I decide to donate a portion' → Is Selected."

**Q3** — Quels sont les 3 réglages critiques à l'export des données Qualtrics pour SPSS ?

> **Réponse** : (1) Use **numeric values**, (2) Recode unanswered as **-99**, (3) Export **viewing order data** for randomized surveys.

**Q4** — Quelle est la différence entre Skip Logic et Display Logic ?

> **Réponse** : **Skip Logic** saute à une autre page/fin de survey (ex. consentement → End if "No"). **Display Logic** affiche ou masque une question sur la même page selon une condition.

**Q5** — Quel type de question Qualtrics pour mesurer "I feel guilty about the situation" sur 5 niveaux (5 items similaires) ?

> **Réponse** : **Matrix Table** — idéal pour plusieurs items avec la même échelle de réponse.
