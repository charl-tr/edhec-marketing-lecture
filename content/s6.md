> **Hook** : Tu as collecté tes données. Maintenant quoi ? Cette session te donne les outils pour transformer un fichier SPSS brut en résultats interprétables — qu'ils viennent d'interviews qualitatives ou d'un questionnaire expérimental quantitatif.

**Idée centrale** : Analyser n'est pas lire ses données en diagonale. C'est un processus systématique : coder le quali, vérifier la fiabilité des scales en quanti, choisir le bon test statistique, et lire les outputs SPSS sans paniquer.

## PARTIE 1 — Analyse qualitative [slides 1-15]

### 1.1 Qu'est-ce que l'analyse qualitative ?

L'analyse qualitative = transformer du texte brut (transcriptions d'interviews, focus groups) en **thèmes interprétables**.

Deux approches principales :

| Approche | Description | Quand l'utiliser |
|---|---|---|
| **Thematic Analysis** (Braun & Clarke, 2006) | Identifier des patterns récurrents dans le texte → thèmes | Exploration ouverte, recherche inductive |
| **Content Analysis** (Krippendorff, 2018) | Compter et catégoriser des éléments de contenu prédéfinis | Comparaisons quantifiées, déductif |

> **💡 Intuition** : L'analyse thématique est plus exploratoire ("quels thèmes émergent ?"). Le content analysis est plus proche d'un comptage systématique ("combien de fois ce mot apparaît ?").

### 1.2 Processus de codage [slides 5-10]

```
Transcription brute
      ↓
  Coding (codes)           ← attribuer des étiquettes à des extraits
      ↓
  Catégories               ← regrouper les codes similaires
      ↓
  Thèmes                   ← abstraction supérieure
      ↓
  Théorie / Insights       ← réponse à la question de recherche
```

**Déductif** : partir de codes prédéfinis (cadre théorique existant)
**Inductif** : laisser émerger les codes du texte (ground-up)

> **💡 Analogie** : Coder un texte, c'est comme surligner un livre avec un marqueur différent pour chaque idée. Ensuite tu regroupes les surlignages similaires en thèmes.

### 1.3 Inter-coder reliability [slide 11]

Quand deux chercheurs codent indépendamment le même texte : si leurs codes concordent (inter-coder agreement élevé), l'analyse est plus crédible. C'est une forme de triangulation de la subjectivité.

## PARTIE 2 — Préparation des données quantitatives [slides 16-25]

### 2.1 Multi-item scales et compound scores [slides 16-20]

Un construct abstrait (ex. "culpabilité") est mesuré par **plusieurs items** (questions) pour plus de fiabilité.

**Exemple WWF** : "Guilt feelings" = moyenne de 4 items :
- "I feel guilty about the situation depicted in the picture"
- "I regard myself as guilty for this sort of situations"
- "I feel responsible for the situation depicted in the picture"
- "I consider having a part of responsibility for this type of situation"

**Créer le compound score dans SPSS** :
`Transform → Compute Variable → Variable Name = "Guiltfeelings" → MEAN(item1, item2, item3, item4)`

### 2.2 Cronbach's Alpha — fiabilité de la scale [slides 21-25]

Avant de créer un compound score, il faut vérifier que les items mesurent bien la même chose.

**Dans SPSS** : `Analyze → Scale → Reliability Analysis → Alpha`

| Valeur α | Interprétation |
|---|---|
| α ≥ 0.9 | Excellent |
| α ≥ 0.8 | Good |
| α ≥ 0.7 | Acceptable ✅ (seuil standard) |
| α ≥ 0.6 | Questionable |
| α < 0.6 | Poor → revoir les items |

> **💡 Intuition** : Cronbach's α teste si tes items "rament dans le même sens". Si tu mesures la culpabilité avec 4 questions qui se ressemblent, elles doivent être corrélées. α < 0.7 = les items ne mesurent pas vraiment la même chose.

> **⚠️ Règle** : Toujours vérifier α **avant** de créer le compound score. Ne pas combiner des items si α < 0.7.

## PARTIE 3 — Analyse quantitative avec SPSS [slides 26-76]

### 3.1 Ouvrir les données dans SPSS [slide 27]

`File → Open → Data → [sélectionner le fichier .sav]`

**Data View** : chaque ligne = 1 participant, chaque colonne = 1 variable
**Variable View** : définit les métadonnées de chaque variable

| Colonne Variable View | Signification |
|---|---|
| Name | Nom de code (ex. "Condition") |
| Type | Numérique, String… |
| Label | Nom lisible (ex. "Campaign condition") |
| Values | Value labels (1=Panda, 2=Deer, 3=Rain Forest, 4=Rhino) |
| Measure | Scale / Ordinal / **Nominal** |

> **💡 Intuition** : Variable View = le "dictionnaire" de ton dataset. Sans value labels, SPSS affiche "1, 2, 3, 4" au lieu de "Panda, Deer, Rain Forest, Rhino" → impossible à interpréter.

### 3.2 Choisir le bon test statistique [slide 30]

**La règle d'or** : le test dépend du type de tes variables (IV et DV).

| IV | DV | Test |
|---|---|---|
| Nominale (2 groupes) | Métrique | **Independent t-test** |
| Nominale (3+ groupes) | Métrique | **One-Way ANOVA** |
| Métrique | Métrique | **Linear Regression** |
| Nominale | Nominale | Chi-square |

**Cas WWF** :
- IV = Condition/Campagne (Nominale, 4 groupes) → DV = Culpabilité/Générosité (Métrique) → **ANOVA**
- IV = Genre (Nominale, 2 groupes) → DV = Don (Métrique) → **t-test ou ANOVA** (équivalent)
- IV = Âge (Métrique) → DV = Don (Métrique) → **Régression linéaire**

### 3.3 p-valeur et significativité statistique [slides 28-29]

**H0 (null hypothesis)** : pas de différence entre les groupes / pas d'effet
**H1 (alternative)** : il y a une différence / un effet

| p-valeur | Décision |
|---|---|
| **p < 0.05** | Rejeter H0 → effet **significatif** ✅ |
| **p ≥ 0.05** | Conserver H0 → effet **non significatif** |

> **💡 Intuition** : p = probabilité d'obtenir ces résultats si H0 était vraie. p = 0.008 signifie "il y a seulement 0.8% de chances que ça soit dû au hasard" → on rejette H0.

> **⚠️ Piège** : p-valeur dépend de la taille d'échantillon. Plus l'échantillon est grand, plus il est facile d'atteindre p < 0.05 même pour des effets minuscules. Toujours regarder aussi la taille de l'effet (Mean differences).

### 3.4 One-Way ANOVA [slides 31-55]

#### Quand l'utiliser
IV Nominale (3+ groupes) → DV Métrique

**H0** : μ1 = μ2 = μ3 = μ4 (toutes les moyennes sont égales)
**H1** : au moins une moyenne diffère

#### Étapes dans SPSS

```
Step 1 : Analyze → Compare Means → One-Way ANOVA
Step 2 : Setup
         Dependent List : Guiltfeelings, Responsibility, Generosity
         Factor : Visuals (= Campaign condition)
Step 3 : Options → ✅ Descriptive + ✅ Homogeneity of variance test (Levene)
Step 4 : Post Hoc → ✅ Tukey
Step 5 : Click OK
```

#### Vérifier l'hypothèse de Levene avant tout

**Levene's test** vérifie l'égalité des variances entre groupes (hypothèse de l'ANOVA).

| Résultat Levene | Décision |
|---|---|
| p > 0.05 | Variances égales → ✅ ANOVA standard |
| p < 0.05 | Variances inégales → Utiliser **Welch's ANOVA** |

#### Lire l'output ANOVA

```
ANOVA Table (colonnes clés) :
├── Sum of Squares : variabilité totale
├── df : degrés de liberté
├── Mean Square : variabilité moyenne
├── F : statistique du test
└── Sig. : p-valeur → TA VALEUR CLÉ
```

**Règle de décision** :
- Sig. < 0.05 → différence significative entre au moins 2 groupes ✅
- Sig. > 0.05 → aucune différence significative

#### Résultats WWF exemple (illustratifs)

| DV | F | df | p | Significatif ? |
|---|---|---|---|---|
| Guiltfeelings | 4.103 | (3, 116) | .008 | ✅ Oui |
| Responsibility | 1.875 | (3, 116) | .138 | ❌ Non |
| Generosity | 12.235 | (3, 116) | <.001 | ✅ Oui |

> **⚠️ Piège majeur** : L'ANOVA dit "au moins un groupe diffère" mais **pas lequel**. Pour savoir QUELS groupes diffèrent → Post-hoc Tukey.

---

### 3.5 Post-Hoc Tukey HSD [slides 50-58]

**Pourquoi Tukey ?**
- Test post-hoc le plus courant et conservateur
- Compare toutes les paires de groupes (pairwise)
- Ajuste pour les comparaisons multiples (évite les faux positifs)
- HSD = Honestly Significant Difference (Tukey, 1949)

#### Lire l'output Multiple Comparisons

| Colonne | Signification |
|---|---|
| (I) Visuals | Groupe 1 |
| (J) Visuals | Groupe 2 |
| Mean Difference (I-J) | Différence de moyenne entre les deux groupes |
| Sig. | p-valeur de cette comparaison spécifique |

**Règle** : Si Sig. < 0.05 → ces deux groupes sont **significativement différents** ✅ (marqués d'un *)

#### Exemple Guiltfeelings (WWF)

| Comparaison | Mean Diff | Sig. | Conclusion |
|---|---|---|---|
| Panda vs Deer | 0.167 | .929 | Pas de différence |
| Panda vs Rain Forest | -0.733* | .041 | **Différence significative** |
| Deer vs Rain Forest | -0.900* | .007 | **Différence significative** |
| Rain Forest vs Rhino | 0.500 | .265 | Pas de différence |

#### Exemple Generosity (WWF)

| Comparaison | Mean Diff | Sig. | Conclusion |
|---|---|---|---|
| Panda vs Rain Forest | -2.800* | <.001 | **Très significatif** |
| Panda vs Rhino | -2.000* | .003 | **Significatif** |
| Deer vs Rain Forest | -2.733* | <.001 | **Très significatif** |

→ **Rain Forest et Rhino génèrent significativement plus de générosité que Panda et Deer.**

#### Tableau Descriptives (pour interpréter)

| Campagne | N | Générositié M | SD |
|---|---|---|---|
| Panda | 30 | 1.93 | 1.31 |
| Deer | 30 | 2.00 | 1.44 |
| Rain Forest | 30 | **4.73** | 2.82 |
| Rhino | 30 | 3.93 | 2.77 |

> **💡 La méthode complète** : ANOVA (est-ce qu'il y a un effet global ?) + Tukey (quels groupes diffèrent ?) + Descriptives (qui gagne ?). Les 3 combinés = la réponse complète.

### 3.6 ANOVA pour les covariables [slides 59-64]

#### Genre → Générosité (t-test / ANOVA avec 2 groupes)

IV = Genre (Nominale, 2 groupes : Male/Female) → DV = Donation (Métrique) → **t-test indépendant** (ou ANOVA = équivalent)

**Dans SPSS** :
- ANOVA : Analyze → Compare Means → One-Way ANOVA (Factor = Gender)
- OU t-test : Analyze → Compare Means → Independent-Samples T Test (Grouping Variable = Gender, define groups 1, 2)

**Résultats WWF** :

| Groupe | N | M | SD |
|---|---|---|---|
| Male | 59 | 3.02 | 2.29 |
| Female | 61 | 3.28 | 2.69 |

→ ANOVA : F(1, 118) = 0.329, p = .567 → **Non significatif** : le genre n'a pas d'effet sur la générosité dans ce dataset.

### 3.7 Linear Regression [slides 65-73]

#### Quand l'utiliser
**METRIC IV → METRIC DV**

> **💡 Intuition** : La régression répond à "est-ce que X prédit Y ?" et "dans quelle direction / de combien ?"

**Exemple** : Est-ce que l'âge prédit le don ?
- IV = Âge (Métrique, continu)
- DV = Don (Métrique, continu)
→ Linear Regression

#### Étapes dans SPSS

```
Step 1 : Analyze → Regression → Linear
Step 2 : Dependent variable : Donation
         Independent variable(s) : Age
Step 3 : Click OK
```

#### Lire l'output : Model Summary + Coefficients

**Model Summary** :
- **R²** : pourcentage de variance de DV expliqué par l'IV
- Exemple : R² = 0.03 → l'âge explique 3% de la variance des dons

**Coefficients table** :

```
Coefficients(a)
                    B        Sig.
(Constant)        2.760     .008
Age               0.013     .695
```

| Colonne | Signification |
|---|---|
| **B (Unstandardized)** | Changement de DV pour +1 unité d'IV |
| **Sig.** | Est-ce significatif ? (< 0.05 ?) |

**Interprétation** : B = 0.013, p = .695 → l'âge **ne prédit pas significativement** le don.

> **💡 Formule narrative** : "A linear regression showed that age was not a significant predictor of donation amount (β = 0.013, t(118) = 0.393, p = .695)."

> **⚠️ Notes importantes** :
> - R² en sciences sociales est souvent petit (0.02–0.15 = normal)
> - La régression suppose une relation **linéaire**
> - On peut ajouter plusieurs IVs : Age + Gender → Donation (régression multiple)

### 3.8 Comment rédiger les résultats statistiques [slide 73]

**Golden rules of reporting** :
1. Toujours mentionner le **test utilisé**
2. Inclure la **statistique du test** + la **p-valeur**
3. Reporter les **moyennes et SDs** pour les effets significatifs
4. Utiliser des **tableaux et graphiques** pour appuyer le texte

**Templates :**

**ANOVA** :
> "A one-way ANOVA revealed a significant effect of campaign type on guilt feelings, F(3, 116) = 4.103, p = .008."

**Post-hoc** :
> "Tukey post-hoc tests showed that Campaigns 3 and 4 (M = 4.73; M = 3.93) produced significantly higher guilt than Campaigns 1 and 2 (M = 2.0; M = 1.93), p < .01."

**Régression** :
> "A linear regression showed that age did not significantly predict donation amount (β = 0.013, p = .695)."

## ✅ Test toi sur S6

**Q1** — Tu as une IV nominale avec 4 groupes et une DV métrique. Quel test ?

> **Réponse** : **One-Way ANOVA**, suivie d'un **post-hoc Tukey** pour identifier quels groupes diffèrent.

**Q2** — L'ANOVA donne p = .008 pour la culpabilité. Que conclure ?

> **Réponse** : L'effet est **significatif** (p < 0.05). Au moins une campagne génère un niveau de culpabilité différent des autres. Mais ANOVA ne dit pas laquelle → besoin de Tukey.

**Q3** — À quoi sert Cronbach's alpha et quel est le seuil acceptable ?

> **Réponse** : α mesure la **fiabilité interne** d'une scale multi-items (est-ce que les items mesurent la même chose ?). Seuil acceptable : **α ≥ 0.7**.

**Q4** — B = 0.013, p = .695 dans une régression. Comment interpréter ?

> **Réponse** : L'IV **ne prédit pas significativement** la DV (p > 0.05). B = 0.013 signifie qu'une augmentation d'1 an d'âge est associée à +0.013€ de don — mais cet effet n'est pas statistiquement significatif.

**Q5** — Levene's test donne p = .02. Que faire ?

> **Réponse** : p < 0.05 → les variances sont **inégales** → utiliser **Welch's ANOVA** au lieu de l'ANOVA standard.

**Q6** — Quelle est la différence entre Thematic Analysis et Content Analysis ?

> **Réponse** : **Thematic Analysis** (Braun & Clarke) = identifier des thèmes émergents de façon inductive/déductive. **Content Analysis** = compter et catégoriser des éléments de façon plus systématique et quantifiable (Hsieh & Shannon, 2005).
